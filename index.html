Open Digital Rights Enforcement Framework (ODRE): from descriptive to enforceable policies

Andrea Cimmino1, Juan Cano-Benito1, Raúl García-Castro1,
1Ontology Engineering Group, Universidad Politécnica de Madrid
Paper arXiv Java Code Python Code Zenodo

Abstract

From centralised platforms to decentralised ecosystems, like Data Spaces, sharing data has become a paramount challenge. For this reason, the definition of data usage policies has become crucial in these domains, highlighting the necessity of effective policy enforcement mechanisms. The Open Digital Rights Language (ODRL) is a W3C standard ontology designed to describe data usage policies, however, it lacks built-in enforcement capabilities, limiting its practical application.

This paper introduces the Open Digital Rights Enforcement (ODRE) framework, whose goal is to provide ODRL with enforcement capabilities. The ODRE framework proposes a novel approach to express ODRL policies that integrates the descriptive ontology terms of ODRL with other languages that allow behaviour specification, such as dynamic data handling or function evaluation. The framework includes an enforcement algorithm for ODRL policies and two open-source implementations in Python and Java. The ODRE framework is also designed to support future extensions of ODRL to specific domain scenarios. In addition, current limitations of ODRE, ODRL, and current challenges are reported. Finally, to demonstrate the enforcement capabilities of the implementations, their performance, and their extensibility features, several experiments have been carried out with positive results.

Classification of ODRL enforceable policies

The classification of ODRL enforceable policies revolves around the language used to express them. ODRL policies primarily define usage rules, actions, and constraints for data usage. To make these policies enforceable, they are classified into multiple levels based on how they handle dynamic data or complex scenarios. The simplest form, the ∅-Level, relies on descriptive policies that can be interpreted but not enforced. In contrast, A-Level policies are enforceable, defining actions and constraints with interpretable terms. B1-Level policies introduce variables that need to be interpolated dynamically, while B2-Level policies incorporate templated languages for greater flexibility. Finally, C-Level policies handle complex behaviors and extend enforceability with coded languages, like Python or Java.

Each level in this classification system corresponds to a degree of expressiveness, where ∅-Level policies are purely descriptive and C-Level policies allow for advanced behavior through dynamic inputs or external data sources. By classifying policies this way, it becomes easier to adapt ODRL to specific use cases that require complex data handling, like geospatial constraints or time-based access restrictions.

This classification system also provides critical extension points, enabling practitioners to modify and adapt ODRL policies to new domains without changing the core descriptive language. By adding new ontology terms or extending the interpreted language, new functions can be integrated seamlessly, making the policy framework adaptable for different domains.

Enforcement algorithm

The enforcement algorithm is designed to check if the conditions in ODRL policies are met and, if applicable, enforce the related actions. The algorithm begins by reducing policies from higher levels (B1, B2, C) into the A-Level, which is enforceable. For instance, policies containing dynamic variables or external data inputs must first be processed to replace placeholders with actual values. Once reduced, the algorithm examines the policy's constraints and translates them into an interpreted language (e.g., Python or SPARQL).

After processing, the algorithm evaluates whether the conditions (constraints) hold true. If they do, the associated action is performed. This action could range from granting data access to invoking an API. If the policy’s constraints cannot be met, the action is deferred to a third-party for enforcement, ensuring flexibility in implementation. By generalizing this process, the ODRE framework ensures that ODRL policies can be enforced in real-time scenarios, adapting dynamically to different inputs.

Performance

To evaluate the performance of the ODRE framework, two implementations were developed: one in Python and another in Java. These implementations were tested across 24 different ODRL policies, which examined a variety of enforcement scenarios, from simple permission checks to complex geospatial and temporal constraints. The results demonstrated that both implementations could enforce policies within 0.03 seconds.

Notably, the Java implementation generally outperformed the Python version, except in cases where external API requests were involved. This is due to Java’s more optimized handling of streams for network requests. The experiments validated the framework's ability to enforce policies in near-real-time, making it suitable for applications like data marketplaces or federated data analytics where fast decision-making is crucial. Moreover, the performance tests confirm the framework's extensibility, as new policies and ontology terms can be integrated without significant overhead.
